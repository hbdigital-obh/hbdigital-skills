# CLAUDE.md
Guidance for Claude when working in this repository.

## Objective
Help maintain high-quality, production-ready skill documents in `SKILL.md` format.

## Editing rules
- Prefer small, focused edits.
- Keep wording explicit, actionable, and testable.
- Preserve existing structure unless a structural improvement is requested.
- Do not introduce unrelated changes in the same commit.

## Skill authoring checklist
- Clear purpose statement.
- Explicit trigger conditions.
- Step-by-step execution instructions.
- Constraints and non-goals.
- At least one realistic example.

## Quality bar
- Remove vague language.
- Ensure consistent terminology across skills.
- Avoid overlap between skills; split responsibilities cleanly.

## Git workflow
- Stage only intended files.
- Use descriptive commit messages.
- Keep commits logically grouped by change intent.
