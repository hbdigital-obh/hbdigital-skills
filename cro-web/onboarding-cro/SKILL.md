## Contexte conversion HB Digital

| Page / Projet | Objectif conversion | Benchmark cible |
|---|---|---|

## ÉTAPE 0 — Self-healing

| Situation | Comportement |
|---|---|
| Pas accès aux analytics | Faire un audit visuel + structurel basé sur l'URL |
| URL non accessible | Demander screenshot ou description + recommandations génériques |
| Taux de conversion non fourni | Demander "quel est votre taux actuel ?" et continuer |
| Secteur non précisé | Inférer depuis l'URL ou le contexte |

## ÉTAPE 1 — Diagnostic de conversion

### 1.1 Questions de diagnostic

```
1. Quelle est la page/URL à optimiser ?
2. Quel est le taux de conversion actuel ?
   → Si inconnu : installer le tracking en priorité
3. Quelle est la source de trafic principale ?
   → Paid (Meta/Google) / SEO / Email — détermine les attentes du visiteur
4. Quelle est l'action cible ?
   → Lead (formulaire) / Appel / Achat / Inscription
5. Quel est le device dominant ?
   → Mobile ou Desktop — détermine les priorités d'optimisation
```

### 1.2 Modèle LIFT (facteurs de conversion)

**Les 6 facteurs qui influencent la conversion :**

| Facteur | Question clé | Impact |
|---|---|---|
| **Value Proposition** | La promesse est-elle claire en 5 secondes ? | +++++ |
| **Relevance** | La page correspond-elle à l'annonce/email d'origine ? | +++++ |
| **Clarity** | Le visiteur sait-il quoi faire ? (CTA visible) | ++++ |
| **Anxiety** | Quels freins / peurs bloque le visiteur ? | +++ |
| **Distraction** | Y a-t-il des éléments qui détournent de l'action ? | +++ |
| **Urgency** | Y a-t-il une raison d'agir maintenant ? | ++ |

## ÉTAPE 2 — Audit de landing page

### 2.1 Les 8 zones critiques à auditer

**Zone 1 — Above the fold (ce qu'on voit sans scroller)**
- [ ] Headline clair + bénéfice immédiat ?
- [ ] Sous-titre qui répond à "et alors pour moi ?" ?
- [ ] CTA visible sans scroller ?
- [ ] Visuel en rapport avec l'offre ?
- [ ] Aucun menu de navigation distrayant ?

**Zone 2 — Formulaire**
- [ ] Champs minimum requis (max 4-5 pour un lead cold) ?
- [ ] Labels clairs au-dessus des champs ?
- [ ] Pas de CAPTCHA si évitable ?
- [ ] Texte du bouton orienté bénéfice ("Recevoir mon devis") vs générique ("Envoyer") ?
- [ ] Mention RGPD présente et lisible ?

**Zone 3 — Proposition de valeur**
- [ ] 3 bénéfices clés en bullet points ? (au-dessus de la ligne de flottaison si possible)
- [ ] Chiffres concrets (économies, délai, volume) ?
- [ ] Bénéfice vs caractéristique technique ?

**Zone 4 — Preuves sociales**
- [ ] Témoignage(s) client avec prénom + photo idéalement ?
- [ ] Logos de partenaires/clients si B2B ?
- [ ] Note ou étoiles si applicable ?
- [ ] Chiffre clé (X clients, X € économisés, X ans d'expérience) ?

**Zone 5 — Réduction d'anxiété**
- [ ] Mention "sans engagement" / "gratuit" visible si applicable ?
- [ ] Politique de confidentialité accessible ?
- [ ] Promesse de délai de réponse ?
- [ ] Pas de spam / utilisation des données rassurée ?

**Zone 6 — Urgence et déclencheurs**
- [ ] Limitation (places, stock, offre temporaire) si crédible ?
- [ ] Délai de réponse garanti ("Rappel sous 24h") ?
- [ ] Date limite si applicable ?

**Zone 7 — Mobile**
- [ ] Bouton CTA assez grand sur mobile (> 48px) ?
- [ ] Formulaire utilisable avec une main ?
- [ ] Texte lisible sans zoom (min 16px) ?
- [ ] Temps de chargement < 3 secondes ?

**Zone 8 — Cohérence message**
- [ ] Le titre reprend-il le mot-clé ou l'angle de l'annonce d'origine ?
- [ ] Le ton est cohérent entre annonce et LP ?
- [ ] Pas de surprise négative par rapport à la promesse de l'annonce ?

## ÉTAPE 3 — Recommandations priorisées

### 3.1 Matrice Impact/Effort

**Chaque recommandation est classée :**

| Catégorie | Définition | Quand l'implémenter |
|---|---|---|
| 🔴 Quick win | Impact fort, effort faible | Dans les 24h |
| 🟡 Sprint | Impact fort, effort moyen | Cette semaine |
| 🟢 Roadmap | Impact moyen, effort important | Prochaine itération |
| ⚪ Nice to have | Impact faible | À déprogrammer |

### 3.2 Top recommandations par problème détecté

**Problème : Taux de form trop faible (< 3%)**
- Réduire le nombre de champs au strict minimum
- Changer le texte du bouton (bénéfice > action)
- Ajouter une preuve sociale directement au-dessus du formulaire
- Tester un titre plus direct (chiffre + bénéfice)

**Problème : Taux de rebond élevé (> 70%)**
- Vérifier la cohérence message annonce ↔ LP (premier suspect)
- Améliorer la vitesse de chargement
- Retravailler le headline (plus clair, plus direct)
- Ajouter un visuel rassurant above the fold

**Problème : CPL trop élevé sans problème de trafic**
- Optimiser la LP (voir ci-dessus)
- Réduire le friction du formulaire
- Tester une pop-up exit intent
- Ajouter un A/B test sur le CTA

## ÉTAPE 4 — Plan de tests A/B

### 4.1 Priorisation des tests

**Ordre recommandé :**
1. Headline principal (impact maximal, test rapide)
2. Texte CTA (changement minimal, apprentissage rapide)
3. Visuel hero (remplacer par image de personne vs produit)
4. Réduction de champs (supprimer 1 champ et mesurer)
5. Ajout de preuve sociale (avec vs sans)

### 4.2 Protocole de test A/B

```
Test A/B — [Élément testé]

Hypothèse : Si [modification], alors [résultat attendu]
  car [raison comportementale].

Version A (contrôle) : [description]
Version B (variante) : [description]

Métriques :
  Principale : taux de conversion formulaire
  Secondaire : taux de rebond, temps sur page

Durée minimale : [X jours — calculer selon volume]
Taille d'échantillon minimale : [X visiteurs par version]
Seuil de confiance : 95% minimum avant décision

Critère de victoire : Version B > Version A de X% minimum
```

### 4.3 Calcul durée de test

```
Volume quotidien de trafic : N visiteurs/jour
Taux de conversion actuel : C%
Amélioration minimale détectable : 20%
→ Durée recommandée : [formule disponible sur calculateurs online]

Règle simple :
  < 100 visiteurs/jour → A/B test non fiable, aller direct sur les meilleures pratiques
  100-500 visiteurs/jour → 3-4 semaines par test
  > 500 visiteurs/jour → 1-2 semaines par test
```

- **Phase 1 (audit)** : Microsoft Clarity (gratuit, heatmaps + recordings)
- **Phase 2 (tests)** : GA4 + un outil A/B selon budget
- **Data de base** : Google Analytics 4 obligatoire partout

## RÈGLES ABSOLUES

1. Priorité aux quick wins — impact fort, effort faible d'abord
2. Jamais de recommandation sans estimation d'impact
3. Cohérence message annonce ↔ LP — vérifier en premier
4. A/B test : ne tester qu'1 variable à la fois
5. Finir par 1 action précise à implémenter dans les 24h
